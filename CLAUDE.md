## Project Overview

Browser-based tower defense game collection using p5.js (loaded via CDN). No build step, no package manager, no dependencies beyond p5.js. Each game is a self-contained HTML + single JS file pair.

## Running Locally

Serve the root directory with any static file server:
```bash
python3 -m http.server 8000
# or: npx serve .
```
Open `http://localhost:8000`. There are no tests, linting, or build commands.

## Architecture

### Hub Page
`index.html` — game selection landing page. Each game is a styled card linking to its HTML file. To add a game, add a new `<a class="game-card">` element inside `.games-container` and a matching CSS gradient class.

### Game Structure
Each game lives in `games/<name>/` with two files:
- `<name>.html` — loads p5.js from CDN, includes minimal styling, loads the game script
- `<name>.js` — **all** game logic in a single file (2000-3000+ lines)

Games are fully independent — no shared code, no imports between games.

### p5.js Global Mode
All games use p5.js in **global mode** (not instance mode). Required top-level functions:
- `setup()` — canvas creation and state initialization
- `draw()` — main game loop (update + render every frame)
- `mousePressed()` — click handling for UI and placement
- `windowResized()` — canvas resize handling

### Common Patterns Across Games
- **Game state machine**: string-based `gameState` variable (`"menu"`, `"build"`, `"wave"`, `"gameover"`, etc.)
- **Entity arrays**: global arrays `towers[]`/`satellites[]`, `enemies[]`, `projectiles[]`, `particles[]`
- **Entity classes**: each has `update()` and `draw()` methods
- **Game speed**: a `gameSpeed` multiplier applied to all time-dependent values (movement, fire rate, animations)
- **In-canvas UI**: all menus, buttons, HUD, and tooltips drawn directly on the p5.js canvas — no DOM elements
- **Safe removal pattern**: backward iteration (`for (let i = arr.length - 1; i >= 0; i--)`) when removing entities mid-loop
- **Targeting**: `findTarget()` or equivalent finds closest enemy within range
- **Wave system**: sequential enemy spawning with scaling difficulty
- **Particle effects**: `Particle` class for visual feedback (explosions, impacts)

### Medieval Defense (`games/medieval/`)
- Grid-based placement (40px cells) with waypoint pathfinding
- 4 tower types: Archer, Mage, Cannon, Barracks (spawns Soldier units)
- 3 difficulty levels change paths, enemy stats, and economy
- 10 waves, gold economy, tower upgrades (levels 1-3)
- Canvas: responsive (`windowWidth × windowHeight`)

### Galactic Outpost (`games/space/`)
- Orbital mechanics: 3 rings of slots rotating around a central SpaceStation
- 5 satellite types: Laser, Missile, Tesla, Shield, Power Pylon
- Dual economy: credits (building) + energy (operation, generated by Pylons)
- 1000 waves with boss every 10th wave
- Speed controls: 0x (pause), 1x, 5x, 10x
- Canvas: fixed 900×700px

## Adding a New Game

1. Create `games/<name>/<name>.html` and `games/<name>/<name>.js`
2. Use an existing game's HTML as template (loads p5.js CDN, full-viewport canvas)
3. Implement `setup()`, `draw()`, `mousePressed()`, `windowResized()` in the JS file
4. Add a game card to `index.html` inside `.games-container`
5. Update `README.md` with the game's description

See README.md "Adding a New Game" section for detailed templates.
